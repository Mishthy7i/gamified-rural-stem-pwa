It looks like your local Vite development server either stopped running, or it restarted on a different port!

ERR_CONNECTION_REFUSED on localhost:5174 simply means your browser is trying to grab the video and image files from Port 5174, but there is absolutely no active server listening on that specific port right now to hand the files over.

This usually happens due to one of two things:

The Server Stopped: You might have closed the terminal or stopped the npm run dev script entirely.
The Port Shifted: Sometimes if you have multiple terminals open or restart the server, Vite will grab a slightly different port (like localhost:5173 or localhost:5175). If your browser tab is still looking at the old 5174 port, it will be refused.
How to fix it:

Open a fresh terminal inside your stem-rural-pwa directory.
Run npm run dev again.
Check the terminal output—it will tell you the exact new URL (e.g., http://localhost:5173/).
Just click that exact new URL from the terminal to open a fresh tab, and the Videos and Images will stream perfectly!

That's exactly what was happening! Because the Voice API (`window.speechSynthesis`) is actually built directly into the operating system and outside of React, it doesn't automatically know when the React page disappears! 

If a student clicked "Back" super fast while the Avatar was in the middle of reading the 30-second Hindi paragraph, the OS would keep reading it invisibly while they were sitting on the main dashboard, which is super annoying!

**The Fix:**
I have just injected a strict React Lifecycle "Unmount hook" into both Mission 1 and Mission 2:
```typescript
  // Ensure voice absolutely stops if student leaves the page entirely
  React.useEffect(() => {
    return () => {
      stop();
    };
  }, []);
```
This forces the browser to send an immediate global `stop()` signal to the native Speech API the *exact millisecond* the student clicks "Back" or navigates away. The Avatar voice will now shut off perfectly the moment you leave the lesson page!
